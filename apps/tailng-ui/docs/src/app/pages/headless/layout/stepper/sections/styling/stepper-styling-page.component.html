Styling

Headless stepper exposes one library-owned hook on the root. Everything inside that root, including step row classes and state attributes, is authored by you.

Slot and owner hooks

Selector Who owns it Typical use
[data-slot="stepper"] Library Root spacing, orientation, connectors, and wrapper-level layout.
.step / [data-state] You Step row visuals, complete/current/error emphasis, and icon styling.
[aria-current="step"] You Current-step emphasis tied to semantic state instead of presentation-only classes.

CSS starter

Practical guidance

  • Base your visual current-step treatment on aria-current="step" when possible.
  • Keep owner-authored state names consistent across flows so progress visuals stay predictable.
  • Build connectors, badges, and icons inside each step row rather than expecting extra primitive slots.
  • Use the root slot for layout only and keep step-item styling in your own class or attribute contract.
