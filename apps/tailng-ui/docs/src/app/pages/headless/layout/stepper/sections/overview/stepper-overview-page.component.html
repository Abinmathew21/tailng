Stepper overview

Headless stepper is intentionally minimal. It marks the root container with a stable slot hook and leaves step item structure, current-state semantics, and visual treatment to your own markup.

Basic composition

Attach tngStepper to the ordered list or section that owns your progress flow, then author the step items, labels, and state semantics directly in the surrounding markup.

Accessibility baseline

  • Keep the overall flow semantic by using an ordered list when the sequence matters.
  • Expose the active item with aria-current="step" on the current step row or trigger.
  • Include readable text for complete, optional, or error states instead of relying on color alone.
  • If a step is interactive, use real links or buttons inside each item rather than non-semantic containers.
