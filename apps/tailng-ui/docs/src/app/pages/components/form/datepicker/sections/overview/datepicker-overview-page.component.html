Datepicker overview

TailNG Datepicker ships as a compact material-style wrapper with an editable input, keyboard first calendar navigation, adapter-driven formatting, and bounded month/year/day selection.

Imports

Use TngDatepickerComponent for the default wrapper. Reach for createDatepickerController when you need fully custom UI while keeping the same calendar logic and state contract.

Quick start

Headless orchestration

The wrapper defaults to the material drill-down flow. Headless implementations can keep that behavior or replace it entirely by driving the controller view with showYearsPanel(), showMonthsPanel(), and showDaysPanel().

Behavior baseline

  • The input stays editable and commits valid manual changes back into the calendar state.
  • The centered month-year label opens the year panel, then drills into month and day.
  • Arrow navigation continues across day, month, and year boundaries while respecting bounds.
  • minDate and maxDate disable out-of-range days, months, and years.
  • A custom adapter controls both the input text and the overlay period label.
