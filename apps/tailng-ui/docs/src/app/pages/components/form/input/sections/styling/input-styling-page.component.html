Styling contract

Input styling is slot/state driven. Style against stable data attributes instead of internal DOM assumptions.

CSS contracts

Use these selectors as the public styling surface for tngInput and tngInputGroup.

Slot selectors

Selector Scope Purpose
[data-slot="input"] Input/textarea host Base control styles: typography, spacing, background, placeholder.
[data-slot="input-group"] Group container Container shell: border, radius, background, focus container effects.
[data-slot="input-group-leading"] Leading wrapper Prefix icon/text spacing and alignment.
[data-slot="input-group-control"] Control wrapper Primary layout area for the projected input control.
[data-slot="input-group-trailing"] Trailing wrapper Suffix actions, helper text, or status affordances.

State hooks

State attribute Applied on Use case
[data-focused] Group host Focus ring, focused border, and elevated active styling.
[data-invalid] Control and group host Error border/message color system integration.
[data-disabled] Control and group host Muted visuals, cursor changes, and interaction lock styles.
[data-readonly] Control and group host Readonly visual treatment separate from disabled state.
[data-has-leading] Group host Conditional left padding/layout when leading content is present.
[data-has-trailing] Group host Conditional right padding/layout when trailing content is present.

Common semantic tokens

Token Typical usage
--tng-semantic-background-base Input shell background.
--tng-semantic-background-surface Field container or section background.
--tng-semantic-border-subtle Section and field card borders.
--tng-semantic-border-strong Input shell border.
--tng-semantic-focus-ring Focus indicator around grouped input.
--tng-semantic-accent-brand Focused/selected accent color.
--tng-semantic-accent-danger Invalid/error state border or text.
