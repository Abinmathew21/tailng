Styling contract

Input styling is slot/state driven. You can style without relying on internal class names.

Selectors and state hooks

  • Slots: data-slot="input", data-slot="input-group", data-slot="input-group-leading", data-slot="input-group-control", data-slot="input-group-trailing"
  • State hooks: data-disabled, data-invalid, data-readonly, data-focused, data-has-leading, data-has-trailing
  • Common tokens: --tng-semantic-background-surface, --tng-semantic-border-subtle, --tng-semantic-focus-ring, --tng-semantic-accent-brand
