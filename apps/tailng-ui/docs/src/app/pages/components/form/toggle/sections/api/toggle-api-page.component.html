API reference

Toggle exposes a headless primitive directive and a styled component wrapper for fast product integration.

tngToggle (directive)

Attach on native button elements for pressed-button semantics.

Input Type Default Notes
pressed boolean | null | '' | 'true' | 'false' null Controlled pressed value when supplied.
defaultPressed boolean | '' | string false Initial value for uncontrolled mode.
disabled boolean | '' | string false Blocks interaction and applies disabled attributes.
ariaLabel, ariaLabelledby string | null null Accessible naming hooks.
ariaDescribedBy string | null null Links helper or status text.
Output / Attribute When set Purpose
pressedChange On activation Emits next pressed value.
aria-pressed Always Announces pressed/unpressed state.
data-state Always State hook: on or off.
data-disabled When disabled Styling hook for disabled state.
data-focused, data-focus-visible Focus lifecycle Keyboard-aware focus styling hooks.

tng-toggle (styled component)

Wrapper over tngToggle with icon slots, controlled/uncontrolled modes, and optional ControlValueAccessor wiring.

API Type Details
Inputs pressed, defaultPressed, disabled, pressedLabel, unpressedLabel, aria* Supports controlled and uncontrolled toggle flows.
Output pressedChange Emits next pressed value on user activation.
Forms ControlValueAccessor Works with formControl and formControlName.
Model type boolean Pressed state maps directly to boolean form values.
