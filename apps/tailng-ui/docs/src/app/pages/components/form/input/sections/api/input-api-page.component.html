API reference

Inputs in TailNG are primitive-first. Attach directives to native controls and style through slot/state attributes.

tngInput (directive)

Attach to input[tngInput] or textarea[tngInput].

Inputs

ariaLabel

string | null

Reflects to aria-label. Empty/whitespace values are omitted.

ariaLabelledby

string | null

Reflects to aria-labelledby. Empty/whitespace values are omitted.

ariaDescribedBy

string | null

Reflects to aria-describedby. Empty/whitespace values are omitted.

ariaInvalid

boolean | null | '' | 'true' | 'false'

Override invalid state. If null, it falls back to native :invalid when available.

ariaRequired

boolean | null | '' | 'true' | 'false'

If required=true, required ARIA is forced to true.

disabled, readonly, required

boolean | '' | string

Boolean coercion with native attribute reflection and state hooks.

Output attributes (directive adds)

  • Slot: data-slot="input" on the host control.
  • State hooks: data-disabled, data-readonly, data-invalid.
  • ARIA: aria-invalid="true" only when invalid; aria-required="true" when required.
  • ID: keeps consumer id, otherwise generates tng-input-....

tngInputGroup (component)

Wraps a single projected tngInput control with optional tngInputLeading/tngInputTrailing slots.

  • Renders wrappers only when slot content exists.
  • Tracks focus and emits data-focused on group host.
  • Reflects control state as group-level hooks: data-disabled, data-invalid, data-readonly.
  • Contract: expects exactly one projected tngInput control.

Slot directives

  • tngInputLeading adds data-slot="input-leading".
  • tngInputTrailing adds data-slot="input-trailing".
