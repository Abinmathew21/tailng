API reference

Inputs in TailNG are primitive-first. Attach directives to native controls and style through slot/state attributes.

tngInput (directive)

Attach to input[tngInput] or textarea[tngInput].

Inputs

Property Type Default Details
ariaLabel string | null null Reflects to aria-label. Empty/whitespace values are omitted.
ariaLabelledby string | null null Reflects to aria-labelledby. Empty/whitespace values are omitted.
ariaDescribedBy string | null null Reflects to aria-describedby. Empty/whitespace values are omitted.
ariaInvalid boolean | null | '' | 'true' | 'false' null Overrides invalid state. null falls back to native :invalid.
ariaRequired boolean | null | '' | 'true' | 'false' null Sets ARIA required state. If required=true, this is forced true.
disabled boolean | '' | string false Boolean coercion. Reflects native disabled state and adds data-disabled.
readonly boolean | '' | string false Boolean coercion. Reflects native readonly state and adds data-readonly.
required boolean | '' | string false Boolean coercion. Reflects native required state and drives required ARIA output.

Output attributes

Attribute Applied on When set Details
data-slot="input" Host control Always Stable slot marker for styling contract.
data-disabled Host control When disabled State hook for disabled styling.
data-readonly Host control When readonly State hook for readonly styling.
data-invalid Host control When invalid Set from ariaInvalid override or native validity.
aria-invalid="true" Host control When invalid Only emitted when true.
aria-required="true" Host control When required Set when required=true or ariaRequired=true.
id Host control Always Respects consumer id, otherwise auto-generates tng-input-....

tngInputGroup (component)

Wraps a single projected tngInput control with optional tngInputLeading/tngInputTrailing slots.

Feature Type Details
Projected control contract tngInput projection rule Expects exactly one projected tngInput control.
Conditional wrappers Render behavior Leading/control/trailing wrappers render only when matching slot content exists.
--tng-input-inline-gap CSS custom property Controls inline spacing between slot wrappers and the projected input inside the group shell.
data-focused State attribute Applied on group host while any projected element is focused.
data-disabled, data-invalid, data-readonly Mirrored state attributes Reflected from the projected control onto group host for container-level styling.

Slot directives

Directive Applies to Adds Details
tngInputLeading Projected leading element data-slot="input-leading" Marks prefix/adornment content before the control.
tngInputTrailing Projected trailing element data-slot="input-trailing" Marks suffix/action content after the control.
