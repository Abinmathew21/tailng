API reference

<tng-form-field> owns the shared shell chrome for one projected input[tngInput] and optional slot content.

<tng-form-field> (component)

Use this wrapper when the field needs projected prefix/suffix content or a custom trailing action.

Property Type Default Details
appearance 'outline' | 'solid' | 'ghost' 'outline' Changes the shell presentation while keeping the same slot contract.
size 'sm' | 'md' | 'lg' 'md' Controls the shell height and internal spacing.
tone 'neutral' | 'primary' | 'success' | 'danger' 'neutral' Adjusts shell emphasis and state styling.
fullWidth boolean-like true Controls whether the host stretches to fill its container.

Projected control contract

The shell expects one projected input[tngInput] or textarea[tngInput]. It does not create a native control on its own.

Feature Type Details
Projected control input[tngInput] or textarea[tngInput] The shell mirrors focus, invalid, disabled, and readonly state from the projected input.
data-focused, data-disabled, data-invalid, data-readonly state attrs Applied on the shell host for container-level styling.
--tng-input-inline-gap CSS custom property Controls spacing between prefix, control area, and suffix.

Slot directives

tngPrefix/tngSuffix and tngInputLeading/tngInputTrailing project into the same shell slots.

Directive Applies to Adds Alias Details
tngPrefix Projected leading element data-slot="input-leading" tngInputLeading Marks decorative or semantic content before the input.
tngSuffix Projected trailing element data-slot="input-trailing" tngInputTrailing Marks helper text, actions, or fixed suffix content after the input.
