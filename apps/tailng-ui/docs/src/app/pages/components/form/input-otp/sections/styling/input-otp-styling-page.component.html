Styling

The component does not expose a separate OTP-only token contract. Style it the same way the wrapper does internally: by setting semantic foreground, border, background, and accent tokens on the <tng-input-otp> host.

Theming surface

Your host class is the stable public styling hook. The projected slots inherit the semantic tokens you set there.

Review code

The wrapper keeps the OTP behavior stable while your host class supplies the visual theme.

Payment approval

Apply semantic tokens directly on the wrapper host while keeping the OTP behavior untouched.

Guidance

  • Use host-level semantic tokens for color, border, and focus treatments.
  • Reach for the headless primitive when you need to own the slot markup itself.
  • Keep completion and invalid feedback visible when the code is partially filled.
