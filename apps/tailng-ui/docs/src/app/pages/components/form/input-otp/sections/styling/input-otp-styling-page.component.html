Styling

Style OTP layouts through primitive data attributes and wrapper slot classes. Keep focus, invalid, and completion feedback visible in your product theme.

Recommended hooks

  • [data-slot="input-otp"] for root-level spacing and surface styles.
  • [data-empty], [data-partial], [data-complete] for state feedback.
  • [data-active] and [data-focus-visible] for active-slot highlighting.
  • [data-invalid] + [data-required] for validation affordance.
