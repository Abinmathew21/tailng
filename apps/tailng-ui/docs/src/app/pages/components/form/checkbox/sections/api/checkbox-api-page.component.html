API reference

Checkbox exposes a primitive directive for headless behavior and a styled wrapper with ControlValueAccessor support.

tngCheckbox (directive)

Attach on native input[type="checkbox"].

Input Type Default Notes
checked boolean | '' | string false Marks checkbox checked state.
indeterminate boolean | '' | string false Tri-state mixed mode support.
disabled boolean | '' | string false Blocks interaction and applies disabled attributes.
readonly boolean | '' | string false Keeps focusability while blocking toggles.
required boolean | '' | string false Reflects native required attribute.
invalid boolean | null | '' | 'true' | 'false' null Explicit invalid state override for styling and ARIA.
ariaInvalid boolean | null | '' | 'true' | 'false' null Alternative ARIA invalid override.
name, value string | null null Native form submission support.
ariaDescribedBy string | null null Links helper or error text.
Output Attribute When set Purpose
aria-checked Always Exposes true, false, or mixed.
data-state Always Unified state key: checked | unchecked | mixed.
data-checked, data-unchecked, data-mixed State-specific Convenient styling hooks.
data-disabled, data-readonly, data-invalid When active State hooks for visual contracts.
data-focused, data-focus-visible During focus lifecycle Keyboard/pointer aware focus styling.

tng-checkbox (styled component)

Wrapper over primitive behavior with output events and Angular forms support.

API Type Details
Inputs checked, indeterminate, disabled, readonly, required, invalid, name, value, ariaDescribedBy Forwarded to primitive and root wrapper state.
Outputs checkedChange, indeterminateChange Emits on user changes when not readonly/disabled.
Forms ControlValueAccessor Supports formControl, formControlName, and ngModel.
Model type boolean | 'mixed' Tri-state value can flow through forms for advanced workflows.
