Toast overview

Toast renders short-lived notification messages in a fixed viewport. Use tone, position, and duration to model confirmations, warnings, and persistent error notices.

Imports

Use primitive directives for headless structure or the wrapper component for runtime APIs.

Accessibility baseline

  • Neutral and success tones announce with role="status".
  • Warning and danger tones announce with role="alert".
  • Support keyboard dismissal with Escape and keep a visible close button.
  • Use duration: 0 only for messages that require explicit acknowledgment.
