API reference

Toast ships primitive viewport/item directives and a wrapper component with runtime show/dismiss methods, queue limits, and tone-aware accessibility semantics.

tngToastViewport + tngToastItem primitives

Use primitives when you need full markup control. Tone and open state map to role, live politeness, and visibility attributes.

Primitive input Type Default Notes
[tone] on tngToastItem 'neutral' | 'success' | 'warning' | 'danger' 'neutral' Maps to role, aria-live, and data-tone.
[open] on tngToastItem boolean true Controls hidden and data-state (open/closed).

<tng-toast> wrapper

Wrapper component manages queue state, auto-dismiss timers, Escape dismissal, and emits dismissed when items close.

Wrapper input/output Type Default Notes
[duration] number 4000 Fallback auto-dismiss duration in milliseconds.
[maxVisible] number 4 Keeps only the latest N toasts visible.
[mode] 'toast' | 'snackbar' 'toast' Exposes layout hint via data-mode.
[position] 'top-right' | 'top-left' | 'bottom-right' | 'bottom-left' 'bottom-right' Controls fixed viewport anchor corner.
(dismissed) EventEmitter<string> — Emits the toast id after dismissal.
