Stepper overview

Stepper provides a lightweight structure for ordered progress indicators. Use the headless primitive for complete control or the wrapper component for quick adoption.

Imports

Select the layer that matches your ownership model.

Usage patterns

The primitive only attaches semantics and state hooks. Consumers provide markup, state classes, and connectors.

Accessibility baseline

  • Current step should expose aria-current="step".
  • Step triggers should remain keyboard-focusable in document order.
  • Optional, error, and completed states should be announced in accessible labels.
