Styling contract

The library owns the stepper root hook. State tokens for individual rows are still authored by your app, so keep those conventions centralized if you want current, complete, and error treatment to stay consistent across flows.

State hooks

Hook Where Use
data-slot="stepper" Root (library-owned) Container spacing, orientation layouts, connector styling.
data-state="current" Step item (owner-authored) Active step emphasis and readable current-state styling.
data-state="completed" Step item (owner-authored) Past-step confirmation colors and icon treatment.
data-state="upcoming" Step item (owner-authored) Muted future-step presentation.
data-state="error" Step item (owner-authored) Validation/processing failure styling.

CSS starter

The inner data-state selectors in this starter are conventions from your app, not attributes emitted by the stepper wrapper itself.
