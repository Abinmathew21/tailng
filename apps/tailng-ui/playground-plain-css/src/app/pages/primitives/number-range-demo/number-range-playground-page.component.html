Number Range

Two-value numeric range input. Supports controlled, uncontrolled, reactive-forms, validation, disabled, readonly, and slot-class customisation.

Basic (uncontrolled)

No [value] binding. The component manages its own state. valueChange fires on every input event.

{{ lastBasicEvent() }}

Controlled (signal)

[value] drives the display; (valueChange) updates the signal.

min: {{ priceRange().min ?? 'null' }}  |  max: {{ priceRange().max ?? 'null' }}

Boundaries & step

Configured [min]="0", [max]="10000", [step]="100". Values outside the boundary are reported as invalid but not blocked.

min: {{ boundedRange().min ?? 'null' }}  |  max: {{ boundedRange().max ?? 'null' }}

Placeholders & custom separator

minPlaceholder, maxPlaceholder, and [separator]="'to'" customise the label and divider text.

Reactive forms (formControlName)

Integrates via ControlValueAccessor. The form control value is the full { min, max } object.

form value: {{ form.value }}

{{ form.valid ? 'valid' : 'invalid' }} {{ budgetControl.touched ? 'touched' : 'untouched' }}

Validation states

Range order is validated automatically (min > max = invalid). The [invalid] input forces external invalid state.

min: {{ invalidRange().min ?? 'null' }}  |  max: {{ invalidRange().max ?? 'null' }}

Disabled

Both inputs are disabled and interactions are blocked.

Readonly

Inputs are focusable but value editing is prevented.

Required

[required]="true" sets the native required attribute on both inputs, enabling browser and form validation.

Slot class customisation

The [slot] input accepts a Partial<Record<TngNumberRangeSlots, string>> to attach extra CSS classes to individual parts.

slot.minInput = "custom-input custom-input--min"  ·  slot.separator = "custom-sep"

Accessibility labels

ariaLabel labels the group; minAriaLabel / maxAriaLabel label each input individually. The separator is always aria-hidden.

group: aria-label="Price filter"  ·  min: aria-label="Minimum price in dollars"  ·  max: aria-label="Maximum price in dollars"
